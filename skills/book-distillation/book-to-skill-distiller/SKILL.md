# Book to Skill Distiller — 书籍蒸馏执行方法论

> 将书籍/方法论转化为 Hermes 可调用 Skill 的完整流程
> 核心理论：CommonKADS 知识建模 + 工程控制论系统辨识

---

## 0. 前置判断：这本书值得做 Skill 吗？

**问自己一个问题**：这本书的核心是一个**可以封装复用的控制策略**，还是**一套需要慢慢内化的观察视角**？

| 类型 | 判断标准 | 产出形式 |
|------|---------|---------|
| **ControlStrategy** ✅ | 有明确的"如果...那么..."决策规则，能指导具体行动 | Skill |
| **SystemModel** ✅ | 描述了系统的反馈回路、动态规律 | Skill + 反馈环路图 |
| **SensorUpgrade** ⚠️ | 没有给方法，只是新的观察框架 | 转为记忆卡片，不做 Skill |

**如果答案是"内化"而非"复用"，停止——不要过度建模。**

---

## 1. 第一步：粗读定位（控制论框图）

**目标**：画出这本书的**控制论框图**，定位核心输入→处理→输出

带着这 6 个组件去读：
| 控制论组件 | 你要问的问题 | 提炼为 |
|-----------|-------------|--------|
| **传感器** | 作者提醒注意哪些被忽视的信息？ | 关键观察维度 |
| **比较器** | 作者定义"好/坏"、"成功/失败"的标准是什么？ | 评估准则 |
| **控制器** | 作者给出的决策算法、行动步骤是什么？ | 核心算法 |
| **执行器** | 最终作用于系统的行为是什么？ | 输出动作 |
| **反馈回路** | 如何根据反馈修正？定义了哪些反馈信号？ | 迭代规则 |
| **扰动模型** | 作者识别了哪些常见障碍、陷阱、边界条件？ | 失效边界与反模式 |

**操作**：
1. 翻目录、前言、总结章
2. 用 30 分钟画出一张控制论框图（文字版即可）
3. 确认这本书的 **skill_type**（SystemModel / ControlStrategy / SensorUpgrade）

---

## 2. 第二步：精读提取（CommonKADS 四步法）

### Step 1：任务分解（Task Model）
这本书的方法论可以分解为**不超过 7 个核心任务**。

**操作**：
- 列出每个顶级建议
- 分解为子任务（每个叶子节点 = 一个"如果-那么"规则）
- 用树状结构写出任务层级

### Step 2：推理流程（Inference Model）
完成每个任务，AI 需要执行哪些**推理动作**？

| 推理类型 | 定义 | 示例 |
|---------|------|------|
| **分类** | 将当前情况归类 | "这是一次批评，还是一次请求？" |
| **评估** | 给变量赋予值 | "对方的安全感水平：低/中/高" |
| **规划** | 设计达成目标的步骤 | "先建立信任，再提出敏感问题" |
| **监控** | 检测与期望的偏差 | "我说了'非评判'的话，但对方情绪升级了" |

### Step 3：知识映射（Knowledge Model）
- **静态概念** → 存入 `_memory/` 成为模式卡片
- **操作步骤** → 放入 Skill 的核心算法中
- **关键判断** → 写成"如果-那么"规则

### Step 4：接口设计（Communication Model）
**最小必要输入**（AI 需要从用户那里获取什么才能启动推理）：
1. 触发事件（一句话事实，不是解读）
2. 情绪标记（对方/自己最明显的情绪）
3. 关系语境（权力距离、信任基础）
4. 先前尝试（已经试过什么，效果如何）

## 4. 质量标准（最低要求）

| 必填项 | 说明 |
|--------|------|
| `skill_type` | 必须在 frontmatter 中标注 |
| 核心算法 | 必须有可执行的"如果-那么"规则，不能是摘要 |
| 触发条件 | 必须有明确的关键词、情境、抑制条件 |
| 失效边界 | 必须诚实标注方法论的失效条件 |
| 输入接口 | 必须定义最小必要信息 |
| 控制论映射 | 必须关联到已有记忆卡片 |

**没有失效边界的 Skill 是危险的。没有触发条件的 Skill 是沉睡的。**

---

## 5. 常见陷阱

1. **把摘要当成 Skill**：写了一篇.book 摘要而不是决策规则
2. **过度建模**：一本书能产生 7 个 Skill，其实只需要 1 个
3. **缺少抑制条件**：没说清楚什么时候不能用，比没用更危险
4. **输入贪多**：要求 10 个输入字段，用户根本填不完
5. **忘记溯源**：核心规则没有标注是"原文"还是"推论"

## 6. 执行检查清单

- [ ] **前置判断**：确认是 ControlStrategy/SystemModel，不是 SensorUpgrade
- [ ] **控制论框图**：画出了输入→决策→输出→反馈的完整环路
- [ ] **任务分解**：不超过 7 个核心任务，叶子节点是可执行规则
- [ ] **推理流程**：每个任务有对应的推理类型（分类/评估/规划/监控）
- [ ] **核心算法**：用"如果-那么"写出，不是一段摘要
- [ ] **触发条件**：有 3+ 关键词 + 情境描述 + 明确的抑制条件
- [ ] **失效边界**：诚实标注了 2+ 个失效条件和 2+ 个常见误用
- [ ] **控制论映射**：关联到已有的记忆卡片或控制论准则
- [ ] **格式验证**：通过 hermes-agent-skill-authoring 的 frontmatter 规范
