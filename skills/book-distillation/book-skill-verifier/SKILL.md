# Book Skill Verifier — 书本 Skill 验收方法论

> 检验书籍蒸馏 Skill 是否成功、准确、有用的完整体系
> 三层次：结构完整性 → 内容准确性 → 效用性
> 高阶：与已有认知体系的谐振检验

---

## 概览：验收闭环

```
创建Skill
    ↓
[层次一：结构完整性] → 通过？
    ↓ 是
[层次二：内容准确性] → 通过？
    ↓ 是
[层次二补充：边界压力测试] → 通过？
    ↓ 是
[层次三：效用性检验] → 通过？
    ↓ 是
[高阶：谐振检验] → 关联到记忆层？
    ↓
通过验收 → 正式上线
    ↓ 未通过
修订或归档
```

---

## 层次一：结构完整性（形式验收）

对照清单逐项打勾。

### 组件完备性检查

| 检查项 | 要求 | 重要性 |
|--------|------|--------|
| `skill_type` 定义 | frontmatter 中有 SystemModel / ControlStrategy / SensorUpgrade | 必须 |
| 核心算法 | 有可执行的决策树或"如果-那么"规则 | 必须 |
| 触发条件 | 有明确的关键词（3+）、情境、问题类型 | 必须 |
| 抑制条件 | 有明确的"什么时候不该用" | 必须 |
| 输入接口 | 有最小必要信息定义（不超过 5 项） | 必须 |
| 输出规范 | 有明确的输出格式要求 | 必须 |
| 失效边界 | 有 2+ 个失效条件和 2+ 个常见误用 | 必须 |
| 控制论映射 | 关联到已有记忆卡片或控制论准则 | 应该 |
| 决策规则溯源 | 核心规则标注了原文依据 vs 推论 | 应该 |

**快速判断**：
- 缺少"失效边界"→ 危险的 Skill
- 缺少"触发条件"→ 沉睡的 Skill
- 两项都缺→ 这个 Skill 还没准备好上线

## 层次二：内容准确性（实质验收）

### 维度 2：可追溯性审查

**操作步骤**：
1. 从 Skill 中随机抽取 3 条核心规则
2. 回到原书对应章节找到依据
3. 判断：
   - **[事实]**：规则在原书有明确原文支撑
   - **[推论]**：规则是提炼者的推断

**通过标准**：至少 2 条有原文支撑，且**[推论]**被明确标注

**追问**：
> "如果原作者看到这条规则，他会点头说'对，这就是我的意思'，还是会皱眉说'这有点过度简化了'？"

### 维度 3：边界条件压力测试

这是最有效的准确性检验。构造三类极限输入：

#### 3.1 边界案例（刚好踩在失效边界上）
**测试**：Skill 能否正确识别"此时不该用我"？

```
输入示例：
"仓库着火了，请用非暴力沟通协调一下"
预期：Skill 识别为紧急情境，提示切换至紧急指令模式
```

#### 3.2 对抗案例（表面有关键词，实际不适用深层逻辑）
**测试**：Skill 会不会被关键词误导？

```
输入示例：
"我觉得你根本不关心这个项目"
预期：Skill 识别为"隐性评判包裹在感受句式里"，不是正常情绪表达
```

#### 3.3 反模式验证（故意触发"常见误用"描述）
**测试**：Skill 能否识别并警示？

```
输入示例：
"我感到你总是迟到，这让我很失望"
预期：Skill 识别"你总是"是泛化评判，指出这是误用而非有效观察
```

**通过标准**：3 类测试中至少 2 类正确识别并响应

## 层次三：效用性检验（使用验收）

### 维度 4：触发敏感性-特异性平衡

**敏感性**（该触发时是否触发）：记录漏触发率
**特异性**（不该触发时是否误触发）：记录误触发率

**操作**：
- 在日常对话中刻意埋入测试：
  - 5 个该触发情境 → 检查激活率（目标 > 80%）
  - 5 个不该触发但含关键词情境 → 检查误激活率（目标 < 20%）

**调整策略**：
- 漏触发率高 → 放宽关键词，加入更多情境描述
- 误触发率高 → 收紧触发条件，加入抑制条件

### 维度 5：输出可操作性检验

**操作**：取 Skill 生成的 5 条建议，问：

> "一个完全不懂这本书的人，读完这条建议后，能否立刻做出一个具体的、不同的动作？"

| 评级 | 标准 |
|------|------|
| ✅ 可操作 | 能说出具体的第一步动作 |
| ⚠️ 部分可操作 | 方向对但不知道怎么做 |
| ❌ 不可操作 | 一堆正确但无用的废话 |

**通过标准**：5 条中至少 4 条达到"可操作"

## 高阶：与已有认知体系的谐振检验

### 维度 6：谐振检验（三问）

**问题 1：记忆层沉淀？**
调用该 Skill 5 次后，`_memory/` 下是否有新增的模式卡片？Skill 的实践有没有产生新的洞察被写回记忆层？

**问题 2：矛盾检测？**
Skill 的核心假设，是否与你已有的某个根深蒂固的模式冲突？

| 情况 | 处理方式 |
|------|---------|
| 有益张力（激发思考） | 保留，在 Skill 中标注矛盾点 |
| 被悄然忽略 | 回到书中重新审视，或修改 Skill |
| 根本冲突 | 降级为记忆卡片，不做 Skill |

**问题 3：迭代迹象？**
一个月后回看，这个 Skill 是否因为实践反馈被修订过？

| 情况 | 判断 |
|------|------|
| 从未修订且使用率低 | 可能是不良症状（沉睡 or 过拟合） |
| 定期修订且使用率高 | 健康信号 |

## 验收决策矩阵

| 结构完整 | 内容准确 | 效用良好 | 谐振正常 | 决策 |
|---------|---------|---------|---------|------|
| ✅ | ✅ | ✅ | ✅ | **立即上线** |
| ✅ | ✅ | ⚠️ | — | **上线，但标记待优化** |
| ✅ | ⚠️ | — | — | **退回精读重做** |
| ❌ | — | — | — | **退回结构重构** |
| ⚠️ | ⚠️ | ⚠️ | ❌ | **降级为记忆卡片** |

---

## 执行检查清单

- [ ] **结构完整**：所有"必须"项全部打勾
- [ ] **可追溯**：3 条核心规则中至少 2 条有原文标注
- [ ] **边界测试**：边界案例/对抗案例/反模式各至少 1 个通过
- [ ] **可操作性**：5 条输出中至少 4 条有具体行动建议
- [ ] **触发平衡**：敏感性和特异性达到目标区间
- [ ] **谐振检验**：有 1+ 个记忆层关联，或有意识地标注了矛盾点
- [ ] **迭代记录**：如果已有使用记录，确认有过修订或有修订计划

## 常见失败模式

| 模式 | 症状 | 修复方向 |
|------|------|---------|
| 摘要型 Skill | 核心算法区是一大段文字而非规则 | 拆解为"如果-那么"规则 |
| 过拟合 Skill | 只在自己验证过的情境有效 | 泛化触发条件和边界 |
| 沉睡 Skill | 创建后从未被触发或修订 | 检查触发条件是否太严格 |
| 危险 Skill | 没有失效边界，用户在任何情境都照搬 | 补充抑制条件和边界 |
