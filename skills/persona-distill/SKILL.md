# 人物蒸馏 Persona Distillation

将任意人物的思维方式蒸馏成可运行的AI Skill——心智模型、认知框架、表达DNA全部保留。

## 核心理念

> "你想蒸馏的下一个员工，何必是同事" — 蒸馏乔布斯、马斯克、芒格、费曼，让他们都给你打工。

不是角色扮演，是提取**认知操作系统**。框架能提取，灵感不能。

---

## 蒸馏5层模型

| 层次 | 说明 | 示例 |
|------|------|------|
| **1. 表达DNA** | 语气、节奏、用词偏好 | Naval用短句+反问；乔布斯用"极度聚焦" |
| **2. 心智模型** | 认知框架 | 马斯克用"第一性原理"；芒格用"逆向思考" |
| **3. 决策启发式** | 判断逻辑 | 张雪峰用"ROI教育观"；特朗普用"谈判极限施压" |
| **4. 反模式** | 什么不做/价值观底线 | 乔布斯不做"足够好"；芒格不投不理解的公司 |
| **5. 诚实边界** | 明确标注做不到什么 | 捕捉不了直觉、灵感；公开表达≠真实想法 |

## 工作流程

### Phase 1: 数据收集

**优先级排序的数据源**：

| 优先级 | 数据源 | 获取方式 | 说明 |
|--------|--------|---------|------|
| P0 | 聊天记录 | 飞书API / 微信SQLite / Slack API | 最真实的日常表达 |
| P1 | 访谈/演讲 | YouTube字幕 / 播客转录 | 深度思考过程 |
| P2 | 书籍/文章 | PDF / 网页抓取 | 系统性框架 |
| P3 | 社交媒体 | Twitter/X / 微博 | 快速判断风格 |
| P4 | 他人描述 | 采访 / 二手资料 | 补充盲区 |

**数据格式要求**：
- 原始文本（.txt / .md / .json）
- 标注来源和时间
- 最小量：10万字（越丰富越准确）

### Phase 2: 调研提炼

对每位蒸馏对象进行系统性调研：

```
调研维度：
├── 核心价值观（最在乎什么）
├── 决策框架（怎么判断对错）
├── 思维模型（用什么认知工具）
├── 表达风格（语气/节奏/用词）
├── 反模式（明确不做什么）
└── 边界（承认做不到什么）
```

**输出：人物认知档案**

### Phase 3: Skill生成

创建Skill仓库结构：

```
persona-skill/
├── SKILL.md              # 核心文件（必须）
├── assets/
│   └── persona.md        # 详细认知档案
├── data/
│   ├── sources.md        # 数据来源清单
│   └── quotes.md         # 核心语录库
├── examples/
│   └──对话示例.md        # 验证对话
└── README.md             # 安装说明
```

### Phase 4: 验证迭代

**验证对话测试**：

```
用户 ❯ [实际问题]
蒸馏人物 ❯ [基于认知框架的回答]

验证点：
✓ 是否用了他/她的思维框架？
✓ 表达风格是否一致？
✓ 是否避免了角色扮演的复读感？
```

```markdown

---

# [人物名] 认知框架

## 核心思维模型
[最重要的3-5个认知框架]

## 决策启发式
[遇到问题时怎么思考]

## 表达DNA
- 语气：[正式/随意/犀利/温和]
- 节奏：[短句/长段落/总分总]
- 用词偏好：[专业术语/大白话/比喻]

## 反模式（不做）
1. [明确不做什么]
2. ...

## 诚实边界
- ❌ 做不到：[直觉/灵感/未公开信息]
- ⚠️ 局限：[截止调研时间]

## 典型对话风格示例

用户 ❯ [问题]
[人物名] ❯ [基于框架的回答]
```

## 快速开始蒸馏流程

当用户提供「蒸馏[人物名]」时：

1. **确认数据源** — 询问用户提供哪些材料（聊天记录/文章/书籍/访谈链接）
2. **收集数据** — 按P0-P4优先级收集原始文本
3. **提炼框架** — 分析认知档案（5层模型）
4. **生成Skill** — 创建SKILL.md和完整仓库
5. **验证对话** — 用测试问题验证风格一致性
6. **交付使用** — 指导用户安装和调用方式

## 🔬 进阶技巧（从其他框架吸收）

### 1. 双轨架构（teammate-skill）

**工作 Skill × 人格 Persona** 分离，效果更好：

| 轨道 | 内容 | 作用 |
|------|------|------|
| **工作轨道** | 技术标准、系统边界、工作流程、评审风格 | 决定「做什么」 |
| **人格轨道** | 5层模型（硬规则→身份→表达→决策→社交风格） | 决定「什么态度」 |

调用时：**人格决定态度 → 工作轨道执行 → 输出带此人声音**

```
用户 ❯ 帮我review这个PR
[人物名] ❯ [基于工作轨道的专业评审] + [基于人格轨道的语气]
```

### 2. 六路并行调研（nuwa-openclaw）

调研阶段6个Agent**并行出发**，交叉验证：

| 维度 | 数据来源 |
|------|---------|
| 著作 | 书籍、长篇文章 |
| 访谈 | 播客、YouTube字幕 |
| 社交媒体 | Twitter/X、微博 |
| 批评者视角 | 反对意见、争议 |
| 关键决策 | 重大选择及理由 |
| 人生时间线 | 背景与成长路径 |

### 3. 三重验证标准（nuwa-openclaw）

一个观点要成为「心智模型」，必须通过：

| 测试 | 标准 | 失败降级 |
|------|------|---------|
| **跨域复现** | 2+个不同领域都出现过 | → 启发式 |
| **预测力** | 能推断此人对新问题的立场 | → 启发式 |
| **辨识度** | 不是通用智慧，是此人特有 | → 丢弃 |

### 4. 对抗性验证（nuwa-openclaw）

| 测试 | 验证方式 |
|------|---------|
| **方向一致性** | 用此人公开回答过的问题测试，回答方向得对 |
| **反向诱导** | 用对立观点试探，不能轻易被带跑 |
| **边界测试** | 问超出专长的问题，应表现合理不确定 |
| **辨识度测试** | 匿名化后让人猜是谁，猜不出说明特征不够鲜明 |

验证失败→**自动回炉重做，最多2次**

### 5. 搜索降级策略（nuwa-openclaw）

| 优先级 | 工具 | 说明 |
|--------|------|------|
| 1 | Tavily | AI优化搜索 |
| 2 | ddgs + trafilatura | 免费方案 |
| 3 | Browser自动化 | JS渲染/需登录页面 |
| 4 | 模型自身知识 | 兜底，标注「无外部验证」 |

### 6. 知名度自适应（nuwa-openclaw）

| 类型 | 一手来源数 | 输出规模 |
|------|-----------|---------|
| **知名人物** | ≥10 | 标准Skill，约4500 tokens |
| **小众人物** | <10 | 加挂关键原文档案，约8000-10000 tokens |

## 📋 蒸馏检查清单

**开始前确认**：
- [ ] 数据源类型（聊天/书籍/访谈/社媒）
- [ ] 数据量级（<10万字→提示质量可能不足）
- [ ] 知名度（知名→标准流程，小众→增强模式）

**数据收集**：
- [ ] 原始文本标注来源和时间
- [ ] 覆盖至少3个数据源类型

**提炼阶段**：
- [ ] 完成5层模型提炼
- [ ] 通过三重验证（跨域/预测力/辨识度）

**验证阶段**：
- [ ] 通过对抗性验证（4项）
- [ ] 标注诚实边界

**输出**：
- [ ] SKILL.md完整
- [ ] 诚实边界标注
- [ ] 使用示例

## 注意事项

⚠️ **诚实边界必须标注**：
- 捕捉不了直觉——框架能提取，灵感不能
- 截止调研时间的快照——之后的想法无法反映
- 公开表达≠真实想法——只能基于公开信息

⚠️ **不做的清单同样重要**：
- 知道一个人不做什么，才能真正理解他
- 比知道他做什么更关键

⚠️ **避免角色扮演复读**：
- 不是模仿语录，是提取认知框架
- 同一个问题，名人用不同框架会给出不同答案

⚠️ **双轨优于单轨**：
- 职场人物优先使用双轨架构
- 人格蒸馏可单独使用人格轨道
