# Automotive PPAP Supplier Audit — 供应商 PPAP 审核指南

> 来源：AIAG PPAP 4th Edition + TI Fluid Systems PPAP Review Process + IATF 16949:2016 + Ford/GM CSR  
> 适用：OEM/Tier-1 审核供应商提交的 PPAP 包，决定批准或拒绝

---

## 触发词

- "审核 PPAP"、"PPAP 评审"
- "供应商 PPAP"、"PPAP 批准"、"PPAP 拒绝"
- "供应商质量"、"SQE"、"供应商开发"
- "PPAP 不符合"、"PPAP 缺陷"、"PPAP 纠正措施"
- "供应商 8D"、"供应商 CAPA"

---

## 审核员角色定位

**你是守门员，不是保姆。**

- 你的职责：**判断供应商的 PPAP 包是否达到批准标准**
- 不是帮供应商做 PPAP，而是客观评估
- 发现问题 → 拒绝 + 要求整改 → 供应商自行改善

---

## 第一步：接收 PPAP 包 — 形式审查（5 分钟）

**问：这份 PPAP 包有没有基本完整性？**

### 1.1 基础信息核对

| 检查项 | 合格标准 |
|--------|---------|
| PSW 有供应商签字 | 供应商质量经理或授权人签字 |
| 零件号与 PO 一致 | 零件号 + 修订版本匹配 |
| PPAP Level 与请求一致 | 供应商提交 = 客户要求（通常 Level 3） |
| 语言 | 英文（按合同要求） |
| 提交日期在截止日期内 | 在客户要求的截止日期前提交 |
| 所有 18 项都包含 | 检查清单逐项确认（不允许"NA"除非合理） |

### 1.2 PSW 基础检查

| PSW 字段 | 检查内容 |
|---------|---------|
| 零件号 | 是否与 PO 一致 |
| 提交等级 | Level 3（默认） |
| 提交原因 | 与实际变更匹配（新品/变更/其他） |
| 供应商签字 | 有签字 + 日期 |
| 客户批准栏 | **空白**（客户填写，不是供应商填） |

### 1.3 快速决定

```
形式审查通过 → 进入详细审核
形式审查失败（缺项/零件号不对/无签字）→ 立即退回，不进入技术审核
```

---

## 第二步：详细技术审核 — 按 18 项逐项检查

### 审核原则

> **每项都要问**：这份文件是不是当前的、完整的、准确的、与零件相关的？

---

### 审核组 1：设计与工程文件（项 1-3）

#### 项 1：设计记录（Design Records）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 版本 | 现行有效版本，与 PO 匹配 | 旧版本 / 供应商自行制作的图纸 |
| 特殊特性（SC） | 有标注，符号与客户要求一致 | 未标注 / 符号错误 |
| 工程批准 | 有客户或 OEM 工程签字 | 无签字 / 非授权人签字 |
| GD&T | 所有公差清晰可读 | 缺少尺寸、符号不可解读 |

**常见问题**：
- ❌ 供应商用自己的图纸代替客户提供的图纸
- ❌ 图纸版本低于 PO 要求的版本
- ❌ SC 特性漏标

---

#### 项 2：工程变更文件（Engineering Change Documents）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 变更触发 | 有变更时提交了变更批准文件 | 提交由变更触发但无变更批准 |
| 变更内容 | 变更描述清晰，可追溯 | 无变更描述 |
| 批准链条 | 客户工程变更批准 → 供应商确认 | 批准文件与零件不相关 |

---

#### 项 3：工程批准（Customer Engineering Approval）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 偏差处理 | 原型/试产偏差有客户书面批准 | 未经批准擅自偏离图纸 |
| Interim Approval | 有客户签字的临时批准 | 使用未经批准的临时状态 |

---

### 审核组 2：FMEA 与过程规划（项 4-7）

#### 项 4：DFMEA（Design FMEA）

**适用范围**：供应商负责设计时（Responsible Design）才审核此项

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 版本 | 与设计记录版本一致 | 版本老旧，与当前设计不符 |
| RPN 处理 | 高 RPN（>100）有应对措施 | 高 RPN 无应对措施 |
| 与 PFMEA 链接 | DFMEA 输出传递到 PFMEA | DFMEA 和 PFMEA 相互独立 |

**注意**：如果是客户设计零件，此项可标注"不适用（Customer-designed）"。

---

#### 项 5：过程流程图（Process Flow Diagram）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 覆盖全流程 | 从进料 → 生产 → 出货全覆盖 | 缺少进料检验、成品检验等工序 |
| 工序编号 | 编号清晰，与 PFMEA/CP 一致 | 无编号或编号混乱 |
| 特殊工序 | 热处理/焊接/电镀等特殊工序标注 | 特殊工序未标注 |
| 版本 | 当前版本，与生产状态一致 | 旧版本 |

---

#### 项 6：PFMEA（Process FMEA）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 工序编号一致 | **与流程图和控制计划编号完全一致**（关键！） | 编号对不上 |
| RPN 评分 | S/O/D 评分有依据，不是随机填 | 所有 RPN 都是 100（虚假） |
| 高 RPN 应对 | RPN > 100 有明确应对措施 | 无应对措施 |
| 更新状态 | 与当前工艺状态一致 | 旧版本 PFMEA |

**核心检查点**：
```
流程图工序 #10 → PFMEA工序 #10 → 控制计划工序 #10
（三者编号必须完全一致，一处对不上 → 拒绝）
```

---

#### 项 7：控制计划（Control Plan）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 版本 | 量产版本（Production Control Plan） | 只提供了原型/预量产版本 |
| 工序编号 | **与 PFMEA 和流程图编号一致** | 编号不一致 |
| 特殊特性控制 | SC 有对应的测量和控制方法 | SC 未标注或无控制措施 |
| 抽样方案 | 合理（与风险匹配） | 抽样方案与 PFMEA RPN 不匹配 |
| 反应计划 | 异常发生时有明确的处理步骤 | 空白或写"通知工程师" |

---

### 审核组 3：测量与过程能力（项 8-12）

#### 项 8：MSA（Measurement System Analysis）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| GRR 结果 | **GRR < 30%**（< 10% 为优） | GRR > 30% → 测量系统本身不合格 |
| 测量员 | 至少 3 人/测量系统 | 只用 1 人测量 |
| NDC | NDC ≥ 5 | NDC < 5 |
| 设备校准 | 测量设备在校准有效期内 | 设备过期 |

**GRR 判断标准**：
```
GRR < 10%  → 优秀，批准无悬念
10% ≤ GRR < 30% → 有条件批准，要求供应商改善MSA
GRR ≥ 30%  → 拒绝，测量系统不合格，无法信任测量数据
```

---

#### 项 9：尺寸结果（Dimensional Results）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 样本数量 | **≥ 30 件**（来自 PPAP 生产批次） | 样本不足 30 件 |
| 生产条件 | 生产速率、生产工装、实际环境 | 用原型样件代替生产件 |
| 气球号完整 | 每项测量有编号，对应图纸 | 无气球号，无法追溯 |
| Pass/Fail | 每项特性明确判定 | 判定栏空白 |
| 图纸版本 | 测量报告引用图纸版本正确 | 图纸版本不对 |
| 检验员 | 有检验员签字 + 日期 | 空白报告 |

---

#### 项 10：材料/性能试验结果

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 试验项目 | 覆盖客户材料规格要求的全部项目 | 漏做试验项目 |
| 试验标准 | 引用标准正确且在有效期内 | 标准版本过时 |
| 结果判定 | 全部 Pass | 有 Fail 但无说明 |
| 试验室资质 | 内部：17025 或等效；外部：ISO/IEC 17025 认证 | 无资质证明 |

---

#### 项 11：初始过程研究（Initial Process Studies）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| Cpk 值 | **≥ 1.33**（Level 3 及以上要求） | Cpk < 1.33 → 拒绝 |
| 数据来源 | PPAP 生产批次数据 | 用原型数据代替 |
| 数据组数 | ≥ 25 组 | 数据太少 |
| 异常值处理 | 剔除异常值有理由 | 随意剔除数据 |
| Cpk 计算方法 | 方法合理（与客户统计方法一致） | 方法错误导致 Cpk 虚高 |

**Cpk 审核标准**：
```
Cpk ≥ 1.67 → 优秀，通过
1.33 ≤ Cpk < 1.67 → 有条件通过，要求改善计划
Cpk < 1.33 → 拒绝，不具备量产能力
```

---

#### 项 12：合格实验室文件

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 实验室资质 | ISO/IEC 17025 认证或客户认可 | 无资质文件 |
| 覆盖范围 | 实验室 Scope 覆盖所测项目 | Scope 不包含实测项目 |
| 证书有效期 | 在有效期内 | 证书过期 |

---

### 审核组 4：外观与样本（项 13-16）

#### 项 13：外观批准报告（AAR）

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 有 AAR 表单 | 客户提供的 AAR 表单填写完整 | 无 AAR 表单 |
| 客户代表签字 | 有客户代表目视检查签字 | 供应商自己批准外观 |
| 标准样品 | 有 Master Sample 供目视对比 | 无参考样品 |

---

#### 项 14 & 15 & 16：样本与检查辅具

| 审核点 | 合格 | 不合格 |
|--------|------|--------|
| 样本数量 | 符合客户要求数量 | 数量不够 |
| 样本来源 | PPAP 生产批次（生产工装） | 原型样件 |
| 追溯标签 | 每件有零件号/批次号/日期 | 无标签 |
| Master Sample | 有参考标准样品 | 无 Master Sample |
| 检查辅具 | 专用检具有校准证书 | 无校准证书或过期 |

---

### 审核组 5：客户特殊要求（项 17）

#### 项 17：客户特殊要求（CSR）

| OEM | 常见 CSR 审核重点 |
|-----|----------------|
| Ford | e-PPAP 系统提交、Phased PPAP 执行、PPAP in SAP |
| GM | GMS 要求、BIQ（Built-in Quality）措施落实 |
| Toyota | QDR 自工序完结确认、TPS 实践 |
| VW/Audi | IMDS 提交、REACH/ROHS 符合性 |
| BMW | 供应商手册合规、PPAP 格式 |

**审核方法**：
- 获取该客户的最新 CSR 手册
- 逐项核对 PPAP 包中的 CSR 满足情况
- 不满足 → 记录为 NCR（Non-Conformance Report）

## 第三步：综合判定

### 审核结论选项

| 结论 | 条件 | 后续动作 |
|------|------|---------|
| **PPAP Approved** | 所有关键项合格，无严重不符合项 | 签字批准，通知供应商可以出货 |
| **PPAP Approved with Conditions** | 有非关键不符合项，供应商承诺整改 | 批准 + 整改要求 + 期限 |
| **PPAP Rejected** | 关键项不合格（如 Cpk<1.33 / GRR>30%） | 拒绝，供应商整改后重新提交 |

### 关键不合格项（Critical Issues）—— 直接拒绝

| 不合格项 | 拒绝标准 | 原因 |
|---------|---------|------|
| Cpk | < 1.33 | 过程不具备量产能力 |
| GRR MSA | > 30% | 测量系统不合格，数据不可信 |
| 样本不足 | < 30 件 | 统计依据不足 |
| 三号文件不一致 | PFMEA/流程图/CP 编号对不上 | 系统性错误，说明管理混乱 |
| 实验室无资质 | 无 ISO 17025 | 数据法律效力不足 |
| PSW 无供应商签字 | PSW 空白 | 文件不完整 |
| 零件号/版本不对 | 与 PO 不一致 | 提交的不是客户要的零件 |

### 非关键不符合项（Non-Critical Issues）—— 条件批准

| 不符合项 | 处理方式 |
|---------|---------|
| 签字不完整 | 要求补充签字 |
| 文件版本轻微不一致 | 要求更新到正确版本 |
| 包装/标签轻微不符合 | 要求整改后出货 |
| 缺少 Master Sample（外观件） | 要求提交前先确认外观 |

---

## 第四步：批准后管理

### 批准后关键动作

- [ ] PSW 客户批准栏签字（原件存档）
- [ ] 在客户系统标记 PPAP Approved（如 SAP/QMS）
- [ ] 通知供应商 + 采购 + 物料计划
- [ ] PPAP 文档存档（通常保存 ≥ 15 年）

### 批准后跟踪

| 跟踪项 | 频率 | 目的 |
|--------|------|------|
| 首批出货检验 | 每批 | 确认批量与 PPAP 样件一致 |
| 供应商 PPM 趋势 | 每月 | 确认量产质量维持 |
| 年度 PPAP 复审 | 每年 | 确认持续符合 |
| 变更管理 | 随时 | 有变更立即重新评估 PPAP |

---

## 第五步：供应商 PPAP 拒绝处理

### 拒绝流程

```
收到 PPAP 提交
  → 审核发现关键不符合
  → 填写 PPAP Rejection Notice（含具体不符合项）
  → 发送给供应商质量工程师（SQE）+ 采购
  → 供应商启动整改（通常 5-10 工作日）
  → 供应商重新提交 PPAP
  → 重新审核
```

### PPAP Rejection Notice 模板

```markdown
## PPAP Rejection Notice

**供应商**：[名称]
**零件号**：[零件号]
**PPAP 提交日期**：[日期]
**拒绝日期**：[日期]

### 关键不符合项（Critical）：

| # | PPAP 项 | 不符合描述 | 合格标准 |
|---|---------|-----------|---------|
| 1 | 项 11 Cpk | Cpk = 0.98 | Cpk ≥ 1.33 |
| 2 | 项 6 PFMEA | 工序编号与控制计划不一致 | 三份文件编号一致 |

### 需要供应商提供的整改措施：

| # | 措施 | 责任人 | 截止日期 |
|---|------|-------|---------|
| 1 | 提交 Cpk 改善计划 + 新批次数据 | 李工 | 5 个工作日内 |
| 2 | 重新核对 PFMEA → CP 编号，上传正确版本 | 王工 | 3 个工作日内 |

### 重新提交截止日期：[日期]

### SQE 签字：______________ 日期：______________
```

## 快速审核检查卡（按顺序逐项）

```
形式审查（5 分钟）
  □ PSW 有供应商签字
  □ 零件号 + 版本与 PO 一致
  □ PPAP Level 与要求一致
  □ 截止日期内提交

关键项（直接拒绝）
  □ Cpk ≥ 1.33（项 11）
  □ GRR MSA < 30%（项 8）
  □ 样本 ≥ 30 件（项 9）
  □ PFMEA / 流程图 / 控制计划编号一致（项 5-6-7）
  □ 实验室资质（项 12）
  □ 零件号与 PO 一致（项 1）

非关键项（条件批准）
  □ 签字完整
  □ 文件版本正确
  □ 包装/标签符合要求
  □ Master Sample 有

CSR 审核（按 OEM）
  □ Ford: e-PPAP 提交 + Phased PPAP
  □ GM: GMS / BIQ 符合
  □ Toyota: QDR 自工序完结
  □ VW/Audi: IMDS + REACH

结论判定：
  □ Approved → 签字 + 通知
  □ Approved with Conditions → 批准 + 整改要求
  □ Rejected → 填写 Rejection Notice + 发送
```
