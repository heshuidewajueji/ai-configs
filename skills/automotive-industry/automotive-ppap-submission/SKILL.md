# Automotive PPAP Submission — 供应商 PPAP 提交指南

> 来源：AIAG PPAP 4th Edition + Quality-One + Customiser PPAP Guide + TI Fluid Systems PPAP Process  
> 适用：汽车零部件供应商向 OEM/主机厂/Tier-1 提交 PPAP

---

## 触发词

- "PPAP"、"提交 PPAP"、"PPAP 包"
- "PSW"、"Part Submission Warrant"
- "IATF 16949"、"APQP"、"PPAP 18 项"
- "客户要求 PPAP"、"新零件 PPAP"、"工程变更 PPAP"
- "Cpk"、"MSA"、"GRR"、"FMEA"

## PPAP 是什么（5 分钟搞懂背景）

**PPAP = Production Part Approval Process**  
供应商告诉客户："我具备能力，可以大批量生产了，请批准。"

**什么时候必须提交 PPAP**：

| 触发条件 | 说明 |
|---------|------|
| 新零件首次量产 | 必须提交 |
| 设计/工程变更 | 变更后重新提交 |
| 工艺重大变更 | 新工艺、新工装、新生产线 |
| 材料替代 | 材料变更 |
| 生产中断 ≥12 个月 | 重新激活 |
| 客户要求 | 任何时间客户可要求 |

**不提交的后果**：零件不能出货，客户产线停线 → 罚款 + 赔偿

## 第一步：确认 PPAP 提交要求

**收到 PPAP 请求后，第一件事不是开始做，而是确认要求。**

### 1.1 确认提交级别（Submission Level）

| 级别 | 内容 | 适用场景 |
|------|------|---------|
| **Level 1** | 仅 PSW（零件提交保证书） | 大宗材料、低风险零件 |
| **Level 2** | PSW + 样本 + 有限支持数据 | 成熟零件、已有合作记录 |
| **Level 3** | PSW + **完整**支持数据 | **默认级别（未说明时按此执行）** |
| **Level 4** | PSW + 客户定义的其他要求 | 复杂装配、安全件 |
| **Level 5** | 完整包 + **现场审查** | 新供应商、安全件、重大变更 |

> **原则**：客户未指定级别 → 默认 Level 3

### 1.2 确认客户特殊要求（CSR）

每个 OEM 有额外的客户特殊要求：

| OEM | 特殊要求摘要 |
|-----|------------|
| **Ford** | Phased PPAP、e-PPAP 系统提交、Specific PFMEA 要求 |
| **GM** | Global Supplier Quality Manual、制造过程审核（MPA）时间节点 |
| **Stellantis** | Supplier Quality Assurance Manual、特定文档格式要求 |
| **Toyota** | Toyota APQP 清单、EMT 试验报告 |
| **BMW** | 宝马供应商手册、IMDS 要求 |
| **VW/Audi** | Formel Q、Fertigungsprüfungsbericht（MPB） |

### 1.3 确认截止日期

- 收到 PPAP 请求 → 确认 Submit Date（提交日期）
- PPAP 是 APQP 的最终输出 → 检查 APQP 里程碑是否在轨道上
- **典型周期**：APQP 启动到 PPAP 提交 = 6-18 个月（视项目复杂度）

### 第一步产出
- [ ] 确认 PPAP 提交级别（Level 1-5）
- [ ] 确认客户特殊要求清单（CSR Checklist）
- [ ] 确认 PPAP 提交截止日期
- [ ] 确认提交流道（纸质/e-PPAP 系统/邮件）

## 第二步：组建 PPAP 项目团队

**问：谁来负责这个 PPAP？**

### 核心角色

| 角色 | 职责 | 典型人选 |
|------|------|---------|
| **PPAP Owner** | 整体协调，确保 18 项完整，按时提交 | 项目质量工程师（PQE） |
| **APQP 经理** | APQP 整体进度管控 | 项目经理（PM） |
| **工艺工程师** | PFMEA、控制计划、工艺参数 | 工艺工程师 |
| **测量工程师** | MSA、尺寸测量、GRR | 测量工程师 / QE |
| **实验室对接人** | 材料/性能测试结果协调 | QE 或实验室技术员 |
| **客户质量工程师（CQE）** | 客户沟通、提交、跟进 | 客户质量工程师 |

### PPAP 负责人清单

- [ ] PPAP Owner 指定（通常为 PQE）
- [ ] APQP 会议纪要模板准备
- [ ] 与客户 CQE 建立直接沟通渠道
- [ ] PPAP 进度跟踪表建立（建议用红黄绿灯）

## 第三步：18 项 PPAP 清单（完整版）

### 分类一：设计与工程文件

#### 项 1：设计记录（Design Records）

**要求**：零件图或 3D 数模，**当前有效版本**

| 检查项 | 合格标准 |
|--------|---------|
| 图纸版本 | 与采购订单（PO）版本一致 |
| 特殊特性（SC） | 已用符号标注（客户定义符号） |
| 公差完整 | 所有尺寸 + GD&T 符号可读 |
| 工程签名 | 有工程批准签字或电子签名 |
| 语种 | 英文（客户要求时中英双语） |

**常见错误**：
- ❌ 供应商自己的图纸（不是客户批准的）
- ❌ 图纸版本早于 PO 版本
- ❌ SC 特性未标注

---

#### 项 2：工程变更文件（Engineering Change Documents）

**要求**：若提交由变更触发，提供变更批准文件

- 客户工程变更批准函（Customer ECN）
- SRCA（Supplier Request for Change Approval）批准
- 变更前后对比说明

---

#### 项 3：工程批准（Customer Engineering Approval）

**要求**：客户提供的设计之外的工程批准证据

- 原型阶段偏差批准（Deviation Sheet）
- 临时批准（Interim Approval）证据
- 客户工程邮件批准截图（需存档）

---

### 分类二：FMEA 与过程规划

#### 项 4：设计 FMEA（DFMEA）

**适用范围**：供应商负责设计的零件（"Black Box" 或 "Responsible Design"）

| 检查项 | 合格标准 |
|--------|---------|
| 当前版本 | 与设计记录版本一致 |
| RPN 评估 | 所有高 RPN 项（>100）有应对措施 |
| 链接到 PFMEA | DFMEA 结论已传递到 PFMEA |
| 客户审阅 | 客户需要审阅时，提供证据 |

**注意**：如果是客户设计（Customer-designed part），DFMEA 可能不要求。

---

#### 项 5：过程流程图（Process Flow Diagram）

**要求**：从进料 → 生产 → 出货的全流程可视化

| 检查项 | 合格标准 |
|--------|---------|
| 覆盖全流程 | 包括所有工序（含检验、返工、停放） |
| 工序编号 | **编号与 PFMEA 和控制计划一致**（关键！） |
| 特殊工序标注 | 热处理、焊接、电镀等特殊工序标注 |
| 返工/返修路径 | 明确标注返工流程 |
| 语种 | 英文 |

---

#### 项 6：过程 FMEA（PFMEA）

**要求**：分析每个工序的失效模式及应对措施

| 检查项 | 合格标准 |
|--------|---------|
| 工序编号 | **与流程图和控制计划编号完全一致** |
| RPN 评估 | S（严重度）、O（频度）、D（探测度）评分合理 |
| 高 RPN 处理 | RPN > 100 的项目必须有应对措施 |
| 应对措施落实 | 应对措施已在控制计划中体现 |
| 版本更新 | 与当前工艺状态一致 |

**常见错误**：
- ❌ PFMEA 与控制计划的工序编号对不上
- ❌ RPN 评分虚高（应付了事）
- ❌ 高 RPN 无应对措施

---

### 分类三：控制与测量

#### 项 7：控制计划（Control Plan）

**要求**：定义生产过程如何被控制

| 类型 | 触发条件 |
|------|---------|
| 原型控制计划 | 原型样件阶段 |
| 预量产控制计划 | 试生产阶段 |
| **量产控制计划** | **PPAP 提交时必须提供** |

**检查项**：

| 检查项 | 合格标准 |
|--------|---------|
| 工序编号 | **与 PFMEA 和流程图编号一致** |
| 检测类型 | 100% 检查 vs 抽样检查，理由充分 |
| 测量方法 | 与 MSA 匹配 |
| 反应计划 | 发现异常时的处理步骤明确 |
| 特殊特性控制 | SC 在控制计划中有对应控制措施 |
| 版本 | 当前有效版本 |

---

#### 项 8：测量系统分析（MSA）

**要求**：验证测量系统的能力（Gauge R&R）

| GRR 结果 | 判定 | 处理方式 |
|---------|------|---------|
| **< 10%** | 可接受 | 合格 |
| **10-30%** | 有条件接受 | 需要改善，提交时应说明 |
| **> 30%** | 不可接受 | 必须改善，PPAP 前重做 |

**MSA 最小要求**：
- 至少 **3 个测量员**（1 人重复测量 3 次，或 3 人各测 1 次）
- 测量系统 NDC（Number of Distinct Categories）≥ 5
- 超出量具检定周期 → 结果无效

**测量类型**：
- 连续型数据（尺寸）：用 GRR 研究
- 属性数据（外观检查）：用交叉法（Cross-tabulation）或 Kappa 研究

---

#### 项 9：尺寸结果（Dimensional Results）

**要求**：实际测量报告，证明零件符合图纸

| 要求 | 规格 |
|------|------|
| 最少零件数量 | **30 件**（PPAP 生产批次） |
| 测量环境 | 生产工装、生产速率、实际生产环境 |
| 报告格式 | 客户指定格式（通常为 AIAG 标准格式） |
| 气球号（Balloon Number） | 每项测量有编号，可追溯到图纸 |

**报告必须包含**：
- [ ] 图纸版本
- [ ] 零件编号
- [ ] 零件描述
- [ ] 测量设备（含校准状态）
- [ ] 每项特性的：Balloon 号 + 规格限 + 实测值 + Pass/Fail
- [ ] 检验员签字 + 日期
- [ ] 特殊特性（SC）标注

---

#### 项 10：材料/性能试验结果（Material & Performance Test Results）

**材料试验**（根据材料规格）：

| 试验类型 | 适用零件 | 典型指标 |
|---------|---------|---------|
| 化学成分 | 金属、塑料 | 成分 % |
| 拉伸试验 | 金属件 | 屈服强度、抗拉强度、延伸率 |
| 硬度试验 | 热处理件 | HRC/HV |
| 热膨胀 | 塑料件 | CTE |
| 老化试验 | 橡胶件 | 硬度变化、拉伸保持率 |

**性能试验**（根据功能要求）：

| 试验类型 | 示例 |
|---------|------|
| 功能测试 | 通电测试、软件标定 |
| 耐久性 | 振动试验、疲劳试验 |
| 环境试验 | 温度循环、湿度 |
| 法规试验 | FMVSS、ECE |

**注意**：试验报告必须：
- 注明试验标准及版本（如 ASTM D638 Type I）
- 注明试验设备型号
- 注明试验日期
- 试验室有资质（ISO/IEC 17025 或同等资质）

---

#### 项 11：初始过程研究（Initial Process Studies / SPC）

**目的**：证明生产过程稳定且有能力

| Cpk 值 | 判定 | 处理方式 |
|--------|------|---------|
| **Cpk ≥ 1.67** | 首选 | 优秀，提交无悬念 |
| **1.33 ≤ Cpk < 1.67** | 有条件接受 | 必须提交改善行动计划 |
| **Cpk < 1.33** | 不可接受 | PPAP 会失败，需改善后重新提交 |

**数据要求**：
- 至少 **25 组数据**（每组 ≥ 1 件，推荐 5 件/组）
- 数据来自 **PPAP 生产批次**（生产工装、生产速率）
- 剔除异常值需有理由并记录

**特殊情况**：
- 短周期工艺（< 1 分钟/件）：至少 125 件数据
- 多品种混线：按机型分组提交

---

#### 项 12：合格实验室文件（Qualified Laboratory Documentation）

**要求**：内部实验室或外部实验室资质证明

| 实验室类型 | 资质要求 |
|-----------|---------|
| 内部实验室 | 建立 ISO/IEC 17025 或等效体系 |
| 外部实验室 | **ISO/IEC 17025 认证**，或客户认可 |

**常见错误**：
- ❌ 使用未经认可的内部实验室报告 → PPAP 被拒
- ❌ 实验室范围（Scope）不覆盖所测项目 → 不接受
- ❌ 报告过期

---

### 分类四：外观与样本

#### 项 13：外观批准报告（Appearance Approval Report, AAR）

**适用范围**：有外观要求的零件（颜色、纹理、光泽、表面处理）

**要求**：
- 必须有 **客户代表现场确认**（不能凭照片）
- 客户提供 AAR 表单（如 GM AAR Form）
- 提供标准样品（Standard Sample）供目视参考

---

#### 项 14：生产样品（Sample Production Parts）

| 要求 | 说明 |
|------|------|
| 数量 | 按客户要求（通常 5-10 件） |
| 来源 | PPAP 生产批次（生产工装） |
| 标识 | 零件号 + 批次号 + 日期 |
| 包装 | 客户指定包装方式 |
| 标签 | 含可追溯信息（按客户标签规格） |

---

#### 项 15：标准样品（Master Sample）

**要求**：
- 作为参考标准长期保存
- 标注日期、零件号、版本
- 存放条件：温度/湿度按规定
- **定期复检**（每年至少一次）

---

#### 项 16：检查辅具（Checking Aids）

**要求**：用于检查零件的专用检具

| 类型 | 要求 |
|------|------|
| 专用检具（Check Fixture） | 含校准证书 |
| 硬度块等参考件 | 在有效期内 |
| 影像系统 | 校准记录 |

---

### 分类五：客户特殊要求

#### 项 17：客户特殊要求（Customer-Specific Requirements, CSR）

**要求**：满足客户手册中的额外要求

**常见 CSR 清单**：

| OEM | 常见 CSR |
|-----|---------|
| Ford | e-PPAP 提交、Phased PPAP、材料成分表 |
| GM | GMS（Global Manufacturing System）、BIQ（Built-in Quality） |
| Toyota | QDR（Quality Daily Report）、自工序完pel |
| VW/Audi | Formel Q、IMDS、REACH/ROHS |
| BMW | 供应商手册、PPAP 格式要求 |

---

#### 项 18：散装液体要求（Bulk Fluid Requirements）

**适用范围**：汽车液体（油、冷却液等）供应商

- 通常不适用汽车电子供应商
- 如适用，按客户要求提交

## 第四步：零件提交保证书（PSW）

**PSW = Part Submission Warrant = PPAP 的核心文件**

### PSW 填写要点

| 字段 | 填写说明 |
|------|---------|
| 零件名称 | 按客户图纸 |
| 零件号 | 按 PO |
| 工程变更等级 | 当前工程变更版本 |
| 供应商名称 + 代码 | 按客户供应商代码 |
| 提交等级 | Level X |
| 图纸日期 | 图纸上最新工程日期 |
| 提交原因 | 新品/变更/其他 |
| 变更描述 | 如为变更，描述变更内容 |
| 供应商签字 | 质量经理或授权人 |
| 客户批准签字 | **客户填写**，供应商不填 |

### PSW 关键原则

> **供应商在 PSW 上签字 = 确认提交内容准确**  
> 客户签字 = 正式 PPAP 批准 → 可以出货

### 提交后处理
- [ ] PSW 客户签字原件存档
- [ ] PPAP 批准后，在客户系统中更新状态
- [ ] 将批准信息传达给生产、仓库

---

## 第五步：提交后跟进

### 批准 → 立即出货准备

| 动作 | 说明 |
|------|------|
| 更新 APQP 里程碑 | PPAP Approved 记录日期 |
| 通知生产 | 可以量产 |
| 更新系统 | 客户系统标记 PPAP Approved |
| 批量出货 | 按客户预测/订单 |

### 拒绝 → 快速响应

| 收到拒绝后 | 操作 |
|-----------|------|
| 48h 内分析 | 识别拒绝原因 |
| 8D 启动（必要） | 批量问题启动 8D |
| 重新提交 | 按客户要求的截止日期 |
| 追踪原因 | 防止同类问题再次发生 |

---

## 快速检查清单（提交前必过）

```
□ PPAP Level 确认（默认 Level 3）
□ CSR 清单全部满足
□ 18 项全部完成（无缺项）
□ 图纸版本 = PO 版本
□ 工序编号一致：流程图 = PFMEA = 控制计划
□ MSA GRR < 30%（最好 < 10%）
□ Cpk ≥ 1.33（首选 ≥ 1.67）
□ 实验室资质：ISO 17025 或等效
□ PSW 供应商签字完成
□ 零件从 PPAP 生产批次取出
□ 包装 + 标签符合客户要求
□ 截止日期前提交（提前 5 个工作日提交为佳）
```

---

## 常见 PPAP 拒绝原因

| 排名 | 原因 | 预防措施 |
|------|------|---------|
| 1 | 文件不完整（缺项、版本不对） | 提交前逐项检查清单 |
| 2 | Cpk < 1.33 | SPC 数据提前分析，改善过程能力 |
| 3 | MSA GRR > 30% | 提交前重做 MSA，改善量具或方法 |
| 4 | PFMEA/控制计划编号不一致 | 建立三份文件交叉检查表 |
| 5 | 实验室资质不足 | 使用 ISO 17025 认证实验室 |
| 6 | 尺寸测量样本不足 30 件 | 计划 PPAP 生产批次时规划好数量 |
